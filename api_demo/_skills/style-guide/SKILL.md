# Style Guide Skill

## File Map

- `REFERENCE.md` — complete rules per content type (email, blog, social, support)
- `EXAMPLES.md` — before/after rewrites for the most common mistakes

## Workflow

1. Identify the content type (email, blog post, social post, support reply, etc.)
2. Apply the core brand voice principles below
3. For type-specific rules, read `/skills/style-guide/REFERENCE.md`
4. For before/after examples of common mistakes, read `/skills/style-guide/EXAMPLES.md`
5. Return polished text and briefly explain the key changes made

## Core Principles

- **Voice**: Conversational but authoritative. Write like a knowledgeable friend, not a corporate manual.
- **Tone**: Positive, inclusive, and energetic — never condescending or pushy.
- **Length**: Shorter is better. Cut every word that doesn't earn its place.
- **Person**: Second person ("you/your") for user-facing content; first-person plural ("we/our") for company statements.

## Instant Rules

- No jargon without explanation on first use
- No passive voice unless intentional for tone softening
- Contractions are encouraged ("you're", "it's", "we've")
- Numbers: spell out one–nine; use digits for 10 and above
- Oxford comma always ("red, white, and blue")
- Em dashes for emphasis — use sparingly
- Avoid exclamation marks; let the content be exciting, not the punctuation
